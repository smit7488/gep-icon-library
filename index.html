About the GEP Icon Guide & Generator

What is this tool?

The GEP Icon Guide & Generator is a comprehensive, single-source-of-truth system for the Henry Schein Global eCommerce Platform (GEP) icon library. It consolidates over 1,000 iconsâ€”including Pictographs, UI Icons, and Wireblocksâ€”into one unified, searchable interface that eliminates redundancy and drastically reduces site bloat.

What does it do?

  • Browse & Search: Explore 1,000+ icons with intelligent JSON-based tagging that makes finding the right icon fast and intuitive.
  • Copy-Ready Code: Generate production-ready SVG code instantly with proper class names and sizingâ€”no manual markup required.
  • Color Variations: All icons use currentColor inheritance, meaning one SVG adapts to any brand color via CSS classes. No need for separate color variants.
  • Size Options: Test icons at multiple sizes (16px, 24px, 32px, 48px) to ensure they scale perfectly in your design.
  • Background Patterns: Generate custom wireblock patterns for backgrounds and headers with live preview and color customization.
  • Unified SVG Sprite: All icons live in a single SVG sprite file, eliminating the need to manage thousands of individual files in the DAM.

Why This Tool is Essential

  • Massive Site Optimization: By consolidating all icons into one sprite file, we eliminate the need for thousands of separate image files. This dramatically reduces HTTP requests, page weight, and load times across the entire GEP platform.
  • Single Source of Truth: No more icon hunting across folders, DAMs, or legacy files. Every approved icon lives here, ensuring teams always use the current, standardized version.
  • Zero Redundancy: Traditional icon systems require separate files for each color variant (blue icon, red icon, green icon, etc.). Our system uses one SVG that adapts to any color via CSS, reducing file count by 90%+.
  • Design System Adherence: All icons maintain consistent stroke weights (1.5px), viewBox dimensions, and styling, ensuring perfect alignment with Henry Schein's brand standards.
  • Instant Updates: When an icon is updated or added to the sprite, it's immediately available across all implementationsâ€”no need to re-upload or re-distribute files.
  • Developer Efficiency: Copy complete, production-ready code in one click. No more writing SVG paths by hand or wrestling with file formats.
  • Versatility: Works seamlessly in any environmentâ€”static HTML, React, Angular, Vue, or any framework. The SVG sprite is framework-agnostic.

Technical Architecture

This system is built on a foundation of standardization and automation:

  • Consistent Stroke Weight: All icons maintain a 1.5px stroke width, ensuring visual consistency and crisp rendering at all sizes.
  • JSON-Based Metadata: Icon tags, categories, and relationships are managed through JSON configuration files (icon-tags.json, hs-icons-master-config.json), making the system searchable, scalable, and easy to maintain.
  • Unified Sprite File: Instead of managing 1,000+ individual SVG files for different colors and contexts, all icons are embedded in a single sprite. CSS handles color variations via currentColor, meaning one icon = infinite color possibilities.
  • CSS Color Inheritance: Icons use fill: currentColor and stroke: currentColor, allowing them to inherit color from parent elements or CSS classes. This eliminates the need for separate blue, red, or green versions of the same icon.

Behind the Hood: How It Works

The GEP Icon System is powered by automated Python scripts that ensure consistency, quality, and maintainability:

  • SVG Optimization Scripts: Python automation cleans and optimizes raw SVG files, removing unnecessary metadata, normalizing viewBox dimensions, and standardizing stroke weights to exactly 1.5px.
  • JSON Generation: Scripts automatically generate the configuration files (hs-icons-master-config.json, icon-tags.json) that power the search, filtering, and categorization features.
  • Sprite Compilation: All individual icons are programmatically compiled into a single sprite file, reducing what would be thousands of DAM uploads to just one master file.
  • Tag Management: Metadata tagging is managed through JSON, allowing new icons to be categorized and made searchable without manual indexing or complex database systems.
  • Quality Control: Automated validation ensures every icon meets technical requirements (viewBox, stroke weight, path structure) before being added to the production sprite.

This automation ensures that the icon system remains consistent, scalable, and adheres to strict standardsâ€”eliminating human error and reducing maintenance overhead.

Master CSS System

The tool includes a complete CSS framework for icon styling:

  • Base Classes: .hs-icon sets the default 24px size for UI icons, while .hs-pictograph sets 96px for pictographs. Both include proper display properties and color inheritance.
  • Icon Size Modifiers: .hs-icon-sm (16px), .hs-icon-md (32px), .hs-icon-lg (48px)
  • Pictograph Size Modifiers: .hs-pictograph-sm (64px), .hs-pictograph-md (128px), .hs-pictograph-lg (256px)
  • Pictograph Border: .hs-pictograph-border adds a 1.5pt border using currentColor
  • Color Classes: 30+ color variations across 6 brand colors, each with shade and tint levels. Examples: .hs-blue, .hs-blue-s1 (shade), .hs-blue-t1 through .hs-blue-t4 (tints)
  • Grayscale Options: .hs-dark-gray, .hs-gray, .hs-light-gray-1, .hs-light-gray-2, .hs-black, and .hs-white for neutral applications.

Copy the Master CSS from the header and include it in your project's stylesheet. The CSS is optimized for performance and adheres to GEP standards.
